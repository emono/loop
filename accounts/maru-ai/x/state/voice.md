# X 向け voice / フォーマット: まるくん@AI速報

`account.md` の声を X 用に具体化。速報1本の型と NG を定義。ネタが AI ニュース。
**共通規約（体裁・トーン・品質・免責・鮮度・3state）は `accounts/_family/voice-base.md` を継承**。以下は maru-ai の差分のみ。

## 投稿の型（速報・短〜中尺）

速報なので簡潔に。ネタが濃いときだけスレッドで伸ばす。

- **【速報】** — **1行でシンプルに**（「誰が何を出した/起きた＋日付」だけ）。掴みは短く、誇張しない。
  - 中位モデル云々などの**補足説明は次のツイート（1行）に回す**。速報行に説明を詰め込まない。
- **要点（箇条書き）** — 何が新しいか／主要スペック・数字（事実、出典ベース）。
- **ポイント** — 何がすごい/なぜ重要か（解釈。中立、断定しない）。
- **ソース** — 一次リンク（公式ブログ/論文/リリース）。

## トーン規約（family 共通）
- **ポップで元気・勢いあり**（「キタッ」「出たぞ」「デカい」）、でも正確。**誇張・煽り・ステマはしない**。
- 語尾はタメ口寄りにゆるく（「〜だね」「〜出たぞ」「〜って感じ」）。です/ます で固めすぎない。
- 噂・リークは「噂/未確認」と明示。断定的な性能主張はしない。
- 事実（何が出た）と 解釈（すごさ/重要性）を必ず区別。

## フォーマット規約（family 共通）
- 1ツイート ≤ 140字。長文はスレッド分割（空行区切り、番号ラベル/区切り線なし）。
- **本文はプレーンテキスト**（Markdown 記法を使わない）。**絵文字は使わない**。
- **見出しは【…】**（例 【速報】【料金】【性能】【ポイント】）。`#` はXでハッシュタグ化するので見出しには使わない。見出しに（事実）（解釈）等のメタ注記は付けない。
- **適切に改行して読みやすく**：ダラダラ1段落にしない。
  - 単発1ツイートの目安: 速報行 →（空行）→ ・要点の箇条書き →（空行）→ 結論 → URL。
  - 長い場合はスレッド分割、収まるなら1ツイート内で改行・空行で区切る。
- **定量重視**：スペック・ベンチ・価格は数字で。開示値/実測/試算を区別、**憶測数字は出さない/裏取り必須**。
- **一次ソースの画像添付（推奨）**：公式の図・ベンチ表・スクショがあれば**原本のまま**添付（改変なし）。投稿ディレクトリに同梱（`<slug>/source-*.png`）。

## NG（review でブロック）
- 出典なしの主張・数字。噂の断定。性能の誇張・ステマ。
- 事実と解釈の混同。キャラを盛りすぎて要点がぼける表現。
