# 標準プレスリリース構成（feeed / pr）

`press-release` スキルの draft はこの型に沿う。順序と必須要素を守る。

```
[エンバーゴ（あれば）] 報道関係者各位　◯年◯月◯日　【社名】

# タイトル（結論先出し・約30字）
サブタイトル（補足1行）

【リード文】
誰が・いつ・何を・その意義を1〜2段落で。5W1H が読み取れること。

【背景】
なぜこの発表に至ったか（市場/課題）。

【詳細・特徴】
- 主要ポイント1（事実＋数字）
- 主要ポイント2
- 主要ポイント3

【提供概要】
- 名称 / 提供開始日 / 価格（税込・税抜を明記）/ 対象・提供地域 / 提供方法

【コメント】
「（引用）」——【社名】 ◯◯（役職 氏名）

【画像/図版】image-*.png（キャプション、原本改変なし）

---
■ 会社概要（company.md の boilerplate をそのまま）

■ 本件に関するお問い合わせ先（報道関係者向け）
（必要なら一般のお客様向け窓口も別記）
```

## 使い分け
- 大型発表（新製品ローンチ・資金調達・提携）は**フル構成**で。
- 小さめの告知（受賞・軽微アップデート）は【背景】【コメント】を簡素化可。ただし会社概要・問い合わせ先は必ず残す。
